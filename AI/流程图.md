## Transformer 解码器

以下代码展示了一个标准的 **Transformer 解码器**（即 GPT、GLM 等生成式模型）处理一个输入 Token 并生成下一个 Token 的完整流程。

```mermaid
flowchart TD
    A[输入文本] --> B[分词器 Tokenizer]
    B --> C[词嵌入层 Embedding]
    C --> D[位置编码 Positional Encoding]
    D --> E[进入第 1 层 Transformer 解码器]
    
    E --> F[多头自注意力 Multi-Head Self-Attention]
    F --> G[残差连接 & 层归一化 Add & Norm]
    G --> H[前馈网络 FFN]
    H --> I[残差连接 & 层归一化 Add & Norm]
    I --> J{是否还有下一层?}
    J -->|是| E2[进入下一层]
    E2 --> F2[重复上述过程]
    F2 --> G2[...]
    G2 --> H2[...]
    H2 --> I2[...]
    I2 --> J
    
    J -->|否| K[最终层归一化 Final LayerNorm]
    K --> L[输出投影层 LM Head]
    L --> M[Softmax 转换为概率]
    M --> N[生成下一个 Token]
    N --> O[结束或继续生成]
    
    style A fill:#f9f,stroke:#333
    style N fill:#f9f,stroke:#333
```

> **说明**：每个“层”都包含自注意力、残差连接和前馈网络，整个模型由 N 个这样的层堆叠而成（例如 GLM-5.2 有 70 层）。

### 单个 Transformer 解码器层内部结构（极细粒度）

这张图展示了一个标准层（如 GLM-5.2 的某一层）处理输入向量的完整路径。注意，这里明确拆解了 **残差连接（Add）** 和 **层归一化（Norm）** 的前后顺序（现代模型多采用 Pre-Norm 结构，即先 Norm 再进入子层）。

```mermaid
flowchart TD
    subgraph A[输入张量 X（上一层的输出）]
        X[隐藏状态向量]
    end

    X --> LN1[层归一化 1<br>（LayerNorm）]
    LN1 --> ATTN[多头自注意力机制<br>（详见 图2）]
    
    X --> ADD1[残差连接加法器<br>（Add）]
    ATTN --> ADD1
    ADD1 --> OUT1[中间状态 Z]

    OUT1 --> LN2[层归一化 2<br>（LayerNorm）]
    LN2 --> FFN[前馈网络模块 FFN]
    
    subgraph FFN[前馈网络微观拆解]
        direction TB
        LN2_IN[归一化后的向量] --> GATE[门控线性层<br>（Gate Linear）]
        LN2_IN --> UP[上投影层<br>（Up Projection）]
        GATE --> ACT[激活函数<br>（SiLU / SwiGLU）]
        UP --> MUL[逐元素相乘<br>（Element-wise Multiply）]
        ACT --> MUL
        MUL --> DOWN[下投影层<br>（Down Projection）]
    end

    OUT1 --> ADD2[残差连接加法器<br>（Add）]
    FFN --> ADD2
    ADD2 --> OUT2[本层最终输出向量]

    style X fill:#e1f5fe
    style OUT2 fill:#c8e6c9
```

### 多头自注意力机制微观运算（QKV 全过程）

这是大模型最核心的计算单元。下图拆解了一个“头”（Head）的运算，以及多个头如何合并。对于 **GLM-5.2 的 IndexShare 机制**，你可以理解为：在计算下图中的“缩放点积”时，每隔 4 层共享一个索引矩阵来筛选重要的 Key/Value。

```mermaid
flowchart TD
    subgraph INPUT[输入向量 X]
        X1[输入 X]
    end

    X1 --> PROJ[线性投影层<br>（无偏置）]
    PROJ --> SPLIT[拆分为 Q、K、V 三元组]

    SPLIT --> Q[查询向量 Q]
    SPLIT --> K[键向量 K]
    SPLIT --> V[数值向量 V]

    Q --> MATMUL1[矩阵乘法<br>（Q × K^T）]
    K --> MATMUL1

    MATMUL1 --> SCALE[缩放<br>（除以 √d_k）]
    SCALE --> MASK[因果掩码 Mask<br>（确保只看前面/当前token）]
    MASK --> SOFTMAX[Softmax 归一化<br>（转换为概率权重）]
    SOFTMAX --> WEIGHT[注意力权重矩阵]

    WEIGHT --> MATMUL2[加权矩阵乘法<br>（权重 × V）]
    V --> MATMUL2

    MATMUL2 --> HEAT_OUT[单头注意力输出]

    HEAT_OUT --> CONCAT[拼接所有头<br>（Concat）]
    CONCAT --> OUT_PROJ[最终输出投影层<br>（Output Projection）]
    OUT_PROJ --> FINAL[注意力模块最终输出]

    style Q fill:#bbdefb
    style K fill:#ffecb3
    style V fill:#d1c4e9
    style FINAL fill:#c8e6c9
```

### 图 1：完整大模型（LLM）端到端推理流程图（宏观）
此图覆盖了从用户输入到最终回复生成的**完整自回归循环**，包括 Token 化、位置编码、N 层堆叠、采样策略。

```mermaid
flowchart TD
    Start([用户输入提示词 Prompt]) --> Tokenizer[分词器 Tokenizer<br>（将文本转为 Token IDs）]
    Tokenizer --> Embed[词嵌入层 Embedding<br>+ 旋转位置编码 RoPE]
    Embed --> Loop{自回归循环<br>（生成直到遇到 EOS）}
    
    Loop --> Decoder[堆叠 N 层 Transformer 解码器<br>（每层含 Attn + FFN）]
    Decoder --> LN[最终层归一化 Final LayerNorm]
    LN --> LM[线性输出投影层 LM Head<br>（维度: 隐藏层 → 词表大小）]
    LM --> Softmax[Softmax 计算<br>（生成概率分布）]
    Softmax --> Sample[采样策略<br>（温度缩放 / Top-K / Top-P 截断）]
    Sample --> TokenOut[采样得到当前 Token ID]
    
    TokenOut --> Check{是否满足停止条件?<br>（EOS / 最大长度 / 强制停止）}
    Check -->|否| Append[将 Token 追加到输入序列末尾]
    Append --> Loop
    
    Check -->|是| Detokenize[将生成的 Token IDs 序列解码]
    Detokenize --> Output([流式/完整输出最终文本])

    style Start fill:#e3f2fd
    style Output fill:#c8e6c9
    style Sample fill:#fff9c4
```

### 图 2：GLM-5.2 专用架构流程图（微观层内细节）
此图展示了 GLM-5.2 在处理一个输入序列时的**单层（Layer）内部运算**，并标注了其区别于标准 GPT 的三大核心特征：
- **GLM 混合掩码（Prefix-LM）**：输入被切分为**前缀（Prefix）**和**目标（Target）**，前缀内可双向可见，目标区保持因果自回归。
- **GQA（分组查询注意力）**：为了减少 KV Cache 显存占用，将 Query 分组，每组共享同一个 Key 和 Value 投影。
- **IndexShare 机制**：在每隔 4 层的注意力计算中，共享索引矩阵来筛选重要的 Key/Value（降低计算复杂度）。

```mermaid
flowchart TD
    subgraph GLM5_2[GLM-5.2 单层处理单元]
        Input[输入 Hidden States<br>（上层的输出张量）] --> Norm1[前置 RMSNorm<br>（Pre-Norm 结构）]
        
        %% 混合掩码生成部分
        Input --> MaskGen[生成 GLM 混合注意力掩码矩阵]
        MaskGen --> MaskDetail[① 前缀区域: 全可见双向掩码<br>② 目标区域: 因果掩码<br>③ 前缀→目标: 全可见<br>④ 目标→前缀: 不可见]
        
        Norm1 --> QKV_Proj[线性投影生成 Q, K, V<br>（应用 GQA 分组配置）]
        QKV_Proj --> QKV_Split[将 Q 划分为 Q 分组<br>K 和 V 共享组内投影]
        
        subgraph Attn[分组查询注意力 GQA]
            direction TB
            Qs[各 Query 分组] --> Dot[缩放点积计算<br>（Q_group × K^T / √d）]
            Ks[共享 K 分组] --> Dot
            Dot --> Mask_Apply[应用上述 GLM 混合掩码]
            Mask_Apply --> Soft[Softmax 归一化]
            Soft --> Weighted[加权求和 × 共享 V 分组]
            Vs[共享 V 分组] --> Weighted
            Weighted --> Concat[拼接各分组输出]
        end
        
        %% IndexShare 隔层共享机制
        QKV_Split --> Attn
        Attn --> AttnOut[注意力输出]
        Input --> Add1[残差连接 Add]
        AttnOut --> Add1
        Add1 --> Mid[中间状态 Z]
        
        Mid --> Norm2[前置 RMSNorm]
        Norm2 --> FFN[SwiGLU 前馈网络]
        
        subgraph FFN_detail[SwiGLU 微观拆解]
            direction TB
            IN[归一化向量] --> Up[上投影层 Up-Proj]
            IN --> Gate[门控投影层 Gate-Proj]
            Gate --> SiLU[SiLU 激活函数]
            SiLU --> Mul[逐元素相乘 Element-wise Mul]
            Up --> Mul
            Mul --> Down[下投影层 Down-Proj]
        end
        
        FFN --> FFN_Out[FFN 输出]
        Mid --> Add2[残差连接 Add]
        FFN_Out --> Add2
        Add2 --> LayerOut[本层最终输出]
    end

    %% 跨层 IndexShare 示意（宏观）
    subgraph CrossLayer[跨层 IndexShare 机制]
        direction LR
        L1[第 1 层] -.-> SharePool[索引矩阵共享池<br>（每 4 层共享）]
        L4[第 4 层] -.-> SharePool
        L5[第 5 层] -.-> LocalPool[本地独立索引]
    end
    
    LayerOut --> NextLayer[进入下一层]
    Attn -.-> CrossLayer

    style MaskDetail fill:#ffccbc
    style Attn fill:#b3e5fc
    style FFN_detail fill:#d1c4e9
    style CrossLayer fill:#fff9c4
```

### 核心区别总结（图解补充）

| 维度 | 通用大模型（标准 GPT 类） | GLM-5.2 专用架构 |
| :--- | :--- | :--- |
| **注意力掩码** | 纯因果掩码（上三角全遮） | **混合掩码**：前缀全双向，后缀因果 |
| **注意力变体** | 标准 MHA（多头注意力） | **GQA（分组查询）** + **IndexShare**（隔层共享索引筛选） |
| **前馈网络** | 标准 FFN（ReLU/GELU） | **SwiGLU**（门控线性单元 + SiLU 激活） |
| **位置编码** | 绝对位置 / RoPE | **RoPE**（旋转位置编码），且与掩码策略联动 |
