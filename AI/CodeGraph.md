CodeGraph不是单一的某款软件，而是一个**为AI编程助手（如Claude Code、Cursor）设计的“代码智能导航系统”**。它通过**预索引**的方式，将杂乱无章的代码库转化成一个可以被快速、精准查询的**知识图谱**，从而帮助AI代理从根本上跳出“大海捞针”的低效模式。

### 🧭 核心原理：三步构建“代码地图”

#### 1. 预索引：提前绘制“地图”
CodeGraph不会等到提问时才去阅读文件，而是在项目开始时就将整个代码库“吃透”，构建出一张结构清晰、关系分明的代码知识图谱。

#### 2. 知识图谱建模：定义“城市”要素
在这一步，CodeGraph会将代码逻辑抽象为一张“知识图谱”，主要由两部分构成：

*   **节点：地图上的“地标”**，代表代码中的核心元素：
    *   `File`: 源文件
    *   `Class`: 类
    *   `Function`: 函数
    *   `Module`: 模块
*   **边：连接地标的“路径”**，代表元素之间的逻辑关系：
    *   `CALLS`: 函数调用关系
    *   `IMPORTS`: 模块导入关系
    *   `INHERITS`: 类继承关系
    *   `CONTAINS`: 包含关系（如类包含函数）

#### 3. 智能查询接口：为AI导航
构建好图谱后，CodeGraph会通过一套标准化工具接口（MCP Server），将这些能力抽象成AI可以调用的“高级技能”：

| 工具名称             | 功能描述                                           | 对应传统操作                   |
| -------------------- | -------------------------------------------------- | ------------------------------ |
| `codegraph_context`  | 根据任务描述，返回相关的入口点、符号和代码片段     | `search_content` + `read_file` |
| `codegraph_trace`    | 追踪两个符号之间的完整调用路径                     | 手动逐层阅读代码               |
| `codegraph_explore`  | 批量获取多个文件的源码（行号精确）                 | 多次 `read_file`               |
| `codegraph_callers`  | 找出所有调用了指定函数的调用者                     | `search_content` 搜索函数名    |

### 👷 技术实现：如何高效构建“代码地图”
CodeGraph高效的关键在于其清晰、高效的**技术架构**，主要包含四个核心部分：

*   **代码解析：使用Tree-sitter**：它**不依赖正则表达式去匹配字符串**，而是**真正"读懂"代码**，能精准识别函数、类、方法调用、导入语句等语法结构。它支持超过20种主流编程语言，且具备**增量解析**能力，修改代码后只需更新图谱的一部分，而非全量重建，保证了其运行的高效。

*   **知识图谱存储：使用SQLite**：CodeGraph会将解析出的节点和关系存入一个轻量级、零配置的本地SQLite数据库（`.codegraph/` 目录）中，使所有数据**完全本地化存储**。

*   **图谱构建：依赖关系解析**：CodeGraph会进行**符号解析**，将代码中对函数、类、变量的引用，精准地关联到其在图谱中的定义节点上，从而真正建立起逻辑关系。

*   **查询服务：MCP协议**：通过 Model Context Protocol (MCP) Server 将图谱的查询能力暴露给AI编程助手，让AI可以通过标准的JSON-RPC协议与CodeGraph高效通信。

### 🏆 显著成效：重构AI编程效率
为应对AI代理面临的核心挑战，CodeGraph通过“预索引”机制，实现了**“计算换带宽”**的策略，即通过离线预处理图谱，代替AI在线频繁的文件读写。

这种策略带来的成效显著，官方基准测试数据显示：
*   **Token消耗减少57%**
*   **响应速度提升46%**
