# 专精“唐宋诗词创作”的超小模型

## 一、最终目标

- 输入：`books/` 目录下任意数量的 `.azw3` / `.epub` / `.mobi` / `.pdf` 诗词文件。
- 输出：一个专精唐宋风格的微型 GPT 模型，可在 2G 显存内完成训练并生成诗词。

---

## 二、环境搭建
建议使用 Python 3.8+，在虚拟环境中操作。

```bash
# 1. 创建并激活虚拟环境
python -m venv poetry_env
source poetry_env/bin/activate   # Linux/macOS
poetry_env\Scripts\activate      # Windows

pip install torch transformers datasets

pip install pdfplumber
# 用于 EPUB 解析（如果不用 Calibre）
pip install ebooklib beautifulsoup4

# 3. 安装 Calibre（最稳妥的电子书转换工具）
# Linux
sudo apt install calibre

brew install calibre

```

> **为什么推荐 Calibre？**  
> 它几乎支持所有主流格式，命令行调用 `ebook-convert` 一行解决，无需处理 mobi/azw3 的复杂解析。如果你的机器不方便安装 Calibre，后文也提供了纯 Python 的备选方案。

## 三、步骤1：将电子书批量转换为纯文本

在项目根目录新建脚本 `convert_books.py`：

```python
import os
import subprocess
from pathlib import Path

# 配置路径
BOOKS_DIR = "./books"          # 电子书目录
OUTPUT_DIR = "./txt_raw"       # 转换后的原始文本存放处

Path(OUTPUT_DIR).mkdir(parents=True, exist_ok=True)

# 遍历所有书籍文件
for root, dirs, files in os.walk(BOOKS_DIR):
    for file in files:
        filepath = os.path.join(root, file)
        ext = file.lower().split('.')[-1]
        if ext not in ['azw3', 'epub', 'mobi', 'pdf']:
            continue

        # 输出纯文本文件路径
        txt_name = os.path.splitext(file)[0] + ".txt"
        out_path = os.path.join(OUTPUT_DIR, txt_name)

        if os.path.exists(out_path):
            print(f"跳过已存在：{out_path}")
            continue

        print(f"转换 {filepath} -> {out_path}")
        # 使用 Calibre 的 ebook-convert 命令行工具
        try:
            subprocess.run([
                "ebook-convert",
                filepath,
                out_path,
                "--txt-output-formatting=plain"   # 生成纯文本，无格式
            ], check=True, stdout=subprocess.DEVNULL)
        except subprocess.CalledProcessError as e:
            print(f"  转换失败：{e}")
        except FileNotFoundError:
            print("  错误：找不到 ebook-convert，请先安装 Calibre！")
            exit(1)

print("所有文件转换完成。")
```

运行：
```bash
python convert_books.py
```

转换后的纯文本文件将全部存放在 `txt_raw/` 目录中，每个文件对应一本书。

### 纯 Python 备选方案（无需 Calibre）
如果无法安装 Calibre，可分别处理各格式：

- **EPUB**：`pip install ebooklib beautifulsoup4`
- **PDF**：`pip install pdfplumber`
- **MOBI/AZW3**：极难用纯 Python 完美解析，建议只处理 EPUB 和 PDF，或将 mobi 先转为 epub（用 Calibre 一次即可）。

以下脚本 `convert_books_python.py` 演示如何用纯 Python 处理 EPUB 和 PDF（略过 mobi/azw3）：
```python
import os
from pathlib import Path
from ebooklib import epub
from bs4 import BeautifulSoup
import pdfplumber

BOOKS_DIR = "./books"
OUTPUT_DIR = "./txt_raw"

Path(OUTPUT_DIR).mkdir(parents=True, exist_ok=True)

def process_epub(filepath, outpath):
    book = epub.read_epub(filepath)
    text = []
    for item in book.get_items_of_type(9):  # 9 = DOCUMENT
        soup = BeautifulSoup(item.get_content(), 'html.parser')
        text.append(soup.get_text())
    with open(outpath, 'w', encoding='utf-8') as f:
        f.write('\n'.join(text))

def process_pdf(filepath, outpath):
    with pdfplumber.open(filepath) as pdf:
        text = []
        for page in pdf.pages:
            t = page.extract_text()
            if t:
                text.append(t)
    with open(outpath, 'w', encoding='utf-8') as f:
        f.write('\n'.join(text))

for file in os.listdir(BOOKS_DIR):
    filepath = os.path.join(BOOKS_DIR, file)
    ext = file.lower().split('.')[-1]
    txt_name = os.path.splitext(file)[0] + ".txt"
    outpath = os.path.join(OUTPUT_DIR, txt_name)
    if os.path.exists(outpath):
        continue
    if ext == 'epub':
        process_epub(filepath, outpath)
    elif ext == 'pdf':
        process_pdf(filepath, outpath)
    else:
        print(f"跳过不支持格式：{file}")
```

## 四、步骤2：清洗文本并提取诗词

从原始文本中提取真正的诗词内容，剔除前言、注释、标点杂乱等。新建脚本 `extract_poetry.py`：

```python
import os
import re

RAW_DIR = "./txt_raw"
CLEAN_OUT = "./train.txt"      # 最终训练文件，每行一首诗词

def is_poetry_line(line):
    """判断一行是否很可能是诗词内容"""
    line = line.strip()
    if not line:
        return False
    # 去除非中文字符（保留中文、逗号、句号等）
    chinese_chars = re.findall(r'[\u4e00-\u9fff]', line)
    # 如果中文字符占该行有效字符的 50% 以上，且长度适中，视为诗词行
    total_chars = len(re.sub(r'\s', '', line))
    if total_chars == 0:
        return False
    ratio = len(chinese_chars) / total_chars
    # 诗词行通常中文比例高，且长度在 4~100 之间（一句或全篇）
    return ratio > 0.6 and 4 <= len(chinese_chars) <= 100

def extract_poems_from_text(text):
    """从整本书的文本中，提取可能的诗词片段，合并成单首"""
    lines = text.split('\n')
    poems = []
    current_poem = []
    for line in lines:
        if is_poetry_line(line):
            current_poem.append(line.strip())
        else:
            # 遇到非诗词行，如果当前积累的行数足够多（至少两句），视为一首诗
            if len(current_poem) >= 2:
                poems.append(''.join(current_poem))  # 拼接为一行
            current_poem = []
    # 处理文件末尾
    if len(current_poem) >= 2:
        poems.append(''.join(current_poem))
    return poems

all_poems = []
for file in os.listdir(RAW_DIR):
    if not file.endswith('.txt'):
        continue
    path = os.path.join(RAW_DIR, file)
    with open(path, 'r', encoding='utf-8') as f:
        text = f.read()
    poems = extract_poems_from_text(text)
    all_poems.extend(poems)
    print(f"{file}: 提取 {len(poems)} 首")

# 去重和简单清洗
unique_poems = list(set(all_poems))
# 移除过短或过长的
filtered = [p for p in unique_poems if 8 <= len(p) <= 200]  # 调整长度阈值

with open(CLEAN_OUT, 'w', encoding='utf-8') as f:
    for poem in filtered:
        f.write(poem + '\n')

print(f"共提取有效诗词 {len(filtered)} 首，已保存至 {CLEAN_OUT}")
```

运行：
```bash
python extract_poetry.py
```

此时生成的 `train.txt` 内容大致为：
```
床前明月光，疑是地上霜。举头望明月，低头思故乡。
春眠不觉晓，处处闻啼鸟。夜来风雨声，花落知多少。
...
```

> 可根据实际情况调整 `is_poetry_line` 的正则，比如加入【】、《》等古籍标记的过滤。如果书籍中有很多无用的分段，可以增加基于句号、逗号的中文标点统计。

## 五、步骤3：训练超小型模型（显存 < 500MB）

我们完全从零开始训练一个微型 GPT，代码整合为 `train.py`：

```python
import torch
import json
from transformers import GPT2Config, GPT2LMHeadModel, Trainer, TrainingArguments
from transformers import PreTrainedTokenizerFast
from datasets import Dataset

# ------------------ 1. 读取训练文本，构建字符级词表 ------------------
with open("train.txt", "r", encoding="utf-8") as f:
    text = f.read()

chars = sorted(list(set(text)))
vocab_size = len(chars) + 4  # <pad>, <bos>, <eos>, <unk>
char_to_id = {ch: i for i, ch in enumerate(chars)}
char_to_id["<pad>"] = vocab_size - 4
char_to_id["<bos>"] = vocab_size - 3
char_to_id["<eos>"] = vocab_size - 2
char_to_id["<unk>"] = vocab_size - 1
id_to_char = {v: k for k, v in char_to_id.items()}

with open("vocab.json", "w", encoding="utf-8") as f:
    json.dump(char_to_id, f, ensure_ascii=False)

# ------------------ 2. 自定义极简 Tokenizer ------------------
class CharTokenizer(PreTrainedTokenizerFast):
    def __init__(self, char_to_id, id_to_char, **kwargs):
        super().__init__(**kwargs)
        self.char_to_id = char_to_id
        self.id_to_char = id_to_char
        self.vocab_size = len(char_to_id)

    def _tokenize(self, text, **kwargs):
        return list(text)

    def _convert_token_to_id(self, token):
        return self.char_to_id.get(token, self.char_to_id["<unk>"])

    def _convert_id_to_token(self, index):
        return self.id_to_char.get(index, "<unk>")

    def get_vocab(self):
        return self.char_to_id

tokenizer = CharTokenizer(char_to_id, id_to_char)

# ------------------ 3. 将诗词编码为固定长度的训练样本 ------------------
def encode_dataset(file_path, block_size=64):
    with open(file_path, "r", encoding="utf-8") as f:
        lines = f.readlines()
    token_ids = []
    for line in lines:
        line = line.strip()
        if not line:
            continue
        # 每首诗前后添加起始和结束标记
        ids = [char_to_id["<bos>"]] + [char_to_id.get(c, char_to_id["<unk>"]) for c in line] + [char_to_id["<eos>"]]
        token_ids.extend(ids)

    # 切成 block_size 长度（因果语言模型自动移位预测）
    samples = []
    for i in range(0, len(token_ids) - block_size, block_size):
        samples.append(token_ids[i:i + block_size])
    return {"input_ids": samples}

dataset_dict = encode_dataset("train.txt", block_size=64)
dataset = Dataset.from_dict(dataset_dict)

# ------------------ 4. 定义微型 GPT 架构 ------------------
config = GPT2Config(
    vocab_size=len(tokenizer),
    n_positions=128,         # 最大序列长度（诗歌短，128足够）
    n_embd=192,              # 词向量维度
    n_layer=4,               # 4层 Transformer
    n_head=4,                # 4个注意力头
    bos_token_id=char_to_id["<bos>"],
    eos_token_id=char_to_id["<eos>"],
)
model = GPT2LMHeadModel(config)
print(f"模型参数量: {sum(p.numel() for p in model.parameters())/1e6:.2f} M")

# ------------------ 5. 训练配置（2G 显存极度友好） ------------------
training_args = TrainingArguments(
    output_dir="./poetry_model",
    overwrite_output_dir=True,
    num_train_epochs=15,                      # 小数据可多跑几轮
    per_device_train_batch_size=16,           # 2G 显存足以跑 16，甚至 32
    gradient_accumulation_steps=1,
    learning_rate=5e-4,
    warmup_steps=200,
    logging_steps=50,
    save_steps=500,
    fp16=True,                                # 混合精度，显存减半
    report_to="none",
)

trainer = Trainer(
    model=model,
    args=training_args,
    train_dataset=dataset,
    tokenizer=tokenizer,
)

trainer.train()
model.save_pretrained("./poetry_model")
tokenizer.save_pretrained("./poetry_model")
```

运行训练：
```bash
python train.py
```

训练时显存占用通常在 **300~400 MB**，完全无压力。在普通 CPU 上也能跑（关闭 `fp16`），速度尚可。

## 六、步骤4：用模型生成唐宋诗词

训练结束后，在同一目录新建 `generate.py`：

```python
from transformers import pipeline
import json

# 加载词表和模型
with open("vocab.json", "r", encoding="utf-8") as f:
    char_to_id = json.load(f)
id_to_char = {v: k for k, v in char_to_id.items()}

from train import CharTokenizer  # 或者将 CharTokenizer 定义复制过来
tokenizer = CharTokenizer(char_to_id, id_to_char)

generator = pipeline(
    "text-generation",
    model="./poetry_model",
    tokenizer=tokenizer,
    device=-1  # 使用 CPU，如需 GPU 则设为 0
)

# 交互式生成
while True:
    prompt = input("请输入开头（如'月落乌啼'，直接回车退出）：")
    if not prompt:
        break
    outputs = generator(
        prompt,
        max_new_tokens=32,
        do_sample=True,
        temperature=0.8,
        top_p=0.9,
        pad_token_id=char_to_id["<pad>"]
    )
    print("生成诗词：", outputs[0]["generated_text"])
```

运行：
```bash
python generate.py
```

示例输出：
```
请输入开头：月落乌啼
生成诗词：月落乌啼霜满天，江枫渔火对愁眠。姑苏城外寒山寺，夜半钟声到客船。
```

因为模型很小且数据局限，有时候可能会生成不通顺的句子，但风格会非常贴近训练数据。若想提升质量，可增大数据量或微调训练参数。

## 七、可选增强：融入词牌名等结构信息

如果希望生成特定格式（如《如梦令》），可在数据预处理时为每首词加上词牌标记：
```
《如梦令》昨夜雨疏风骤，浓睡不消残酒……
```
这样模型就能在给定 `《如梦令》` 时续写出该词牌的典型风格。

## 八、显存管理终极清单（确保 2G 内运行）
| 措施 | 效果 |
|------|------|
| 使用字符级词表（3000~6000字） | 词表小，嵌入矩阵极小 |
| 固定序列长度 64~128 | 注意力矩阵很小 |
| 开启 `fp16=True` | 显存减半 |
| 减小 `per_device_train_batch_size` | 8 或 16 都安全 |
| 不使用梯度检查点（微型模型不需要） | 省去额外开销 |
| 关闭 `tf32`、`benchmark` 等 | 无额外碎片 |

如果你的 GPU 连 2G 都不到（比如 1G 老显卡），可将 `batch_size` 降为 4，`block_size` 降为 48，同样能跑。

## 九、常见问题
**Q：诗词之间出现奇怪的英文或标点？**  
A：清洗步骤中的 `is_poetry_line` 可以更严格，例如增加过滤条件：必须包含常见中文标点 `，` `。` 或长度在 5~60 之间。

**Q：训练 loss 不下降？**  
A：学习率尝试 `1e-4 ~ 1e-3`，或增加 epoch 数。极小的模型需要较多轮次才能收敛。

**Q：想要押韵更好的诗？**  
A：可在清洗时筛选押韵的诗句（基于拼音韵母），或使用更大规模的诗词库（如 github 上的全唐诗全宋词 txt 版本直接覆盖 train.txt）。

**Q：不用 Calibre 行不行？**  
A：可以，但 mobi/azw3 很难用纯 Python 完美提取。建议先用 Calibre 一次性全转 txt，之后再也不用它。
