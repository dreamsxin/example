# 大模型的结构

## 📌 适用场景

- 想了解模型的层数、隐藏维度等超参数
- 想对比不同模型的结构差异（如GLM-5.2与Llama 3）
- 想确认模型是否包含特定模块（如MoE、稀疏注意力）
- 想为模型做可视化或调试

## 🧰 前置条件

- 对Python和命令行有基本了解
- 已安装 `transformers`、`torch` 等常用库（方法二需要）
- 网络畅通，能够访问Hugging Face或ModelScope（或已下载模型权重）

---

## 方法一：通过配置文件（config.json）快速查看

这是最直接、无需运行代码的方式。每个在Hugging Face或ModelScope上开源的大模型都会附带一个 `config.json` 文件，其中记录了模型的所有超参数。

### 操作步骤

1. **找到模型的配置文件**  
   - 访问该模型在Hugging Face的页面（如 `https://huggingface.co/THUDM/glm-5.2`），在文件列表中点击 `config.json`。  
   - 如果你已下载模型，直接在本地模型文件夹中找到 `config.json`。

2. **查看关键字段**  
   以下是你应该关注的核心参数及其含义：

| 字段名 | 含义 | 示例（GLM-5.2） |
|--------|------|----------------|
| `num_hidden_layers` | Transformer层的总层数 | 70（推测值） |
| `hidden_size` | 每一层的隐藏维度大小 | 12288（推测值） |
| `num_attention_heads` | 多头注意力的头数 | 96（推测值） |
| `vocab_size` | 词表大小 | 151936（常见值） |
| `intermediate_size` | FFN层（前馈网络）的中间维度 | 49152（推测值） |
| `max_position_embeddings` | 最大位置编码长度 | 1000000（对应1M上下文） |
| `architectures` | 模型使用的类名 | `"GLM5ForCausalLM"` |

   > **注意**：GLM-5.2的创新点之一是 **IndexShare** 机制（每4层稀疏注意力共享一个索引器），这类特殊设计通常不会直接体现在 `config.json` 中，而是需要查阅技术博客或论文。

3. **解读结构全貌**  
   根据这些参数，你就能还原出模型的宏观骨架：一个 `vocab_size` 词嵌入层 → `num_hidden_layers` 个Transformer层（每层包含自注意力、前馈网络、层归一化）→ 最终输出层。

## 方法二：用Python代码打印模型层级

如果你想看到每一层的具体名称、顺序和子模块，可以使用 `transformers` 库加载模型并打印。

### 操作步骤

1. **安装依赖**  
   ```bash
   pip install transformers torch accelerate
   ```

2. **编写并运行脚本**  
   创建一个Python文件，例如 `print_model.py`：

   ```python
   from transformers import AutoModelForCausalLM

   # 替换为你要查看的模型ID（或本地路径）
   model_name = "THUDM/glm-5.2"  # 如果官方已上传，否则用本地路径
   model = AutoModelForCausalLM.from_pretrained(
       model_name,
       torch_dtype="auto",
       device_map="auto"
   )

   # 打印整个模型的结构
   print(model)
   ```

3. **分析输出**  
   控制台会输出类似下面的层级信息（以GLM为例，实际输出会很长）：
   ```
   GLM5ForCausalLM(
     (model): GLM5Model(
       (embed_tokens): Embedding(151936, 12288)
       (layers): ModuleList(
         (0-69): 70 x GLM5DecoderLayer(
           (self_attn): GLM5Attention(
             (q_proj): Linear(12288, 12288, bias=False)
             (k_proj): Linear(12288, 12288, bias=False)
             (v_proj): Linear(12288, 12288, bias=False)
             (o_proj): Linear(12288, 12288, bias=False)
           )
           (mlp): GLM5MLP(
             (gate_proj): Linear(12288, 49152, bias=False)
             (up_proj): Linear(12288, 49152, bias=False)
             (down_proj): Linear(49152, 12288, bias=False)
           )
           (input_layernorm): LayerNorm(12288, eps=1e-5)
           (post_attention_layernorm): LayerNorm(12288, eps=1e-5)
         )
       )
       (norm): LayerNorm(12288, eps=1e-5)
     )
     (lm_head): Linear(12288, 151936, bias=False)
   )
   ```
   这样你就能清晰地看到：
   - 共有70层（0~69）
   - 每层包含自注意力、MLP和两个LayerNorm
   - 注意力投影的权重尺寸等

4. **（进阶）查看特定层**  
   如果你想单独查看某一层（例如第0层的注意力机制），可以在代码中添加：
   ```python
   print(model.model.layers[0].self_attn)
   ```

## 方法三：可视化工具（图形化界面）

如果你觉得文本输出不够直观，可以使用可视化工具生成模型结构图。

### 工具推荐

| 工具 | 优点 | 适用格式 |
|------|------|----------|
| **Netron** | 老牌、轻量、支持多种格式 | ONNX, TensorFlow, Keras, PyTorch（需转换） |
| **TensorKiko** | 直接读取 `.safetensors` / `.pth`，生成交互式HTML | 原生PyTorch权重文件 |
| **torchlens** | 可嵌入Jupyter Notebook，逐层展示 | PyTorch模型 |

### 以Netron为例的操作步骤

1. **将模型转换为ONNX格式**  
   ```python
   import torch
   from transformers import AutoModelForCausalLM

   model = AutoModelForCausalLM.from_pretrained("THUDM/glm-5.2", torch_dtype="auto")
   dummy_input = torch.randint(0, 1000, (1, 10))  # 随机输入

   torch.onnx.export(
       model,
       dummy_input,
       "glm5.onnx",
       opset_version=14,
       input_names=["input_ids"],
       output_names=["logits"],
       dynamic_axes={"input_ids": {0: "batch", 1: "sequence"}}
   )
   ```
   注意：由于模型很大，转换可能需要较多内存和时间，建议在性能较好的机器上进行。

2. **用Netron打开ONNX文件**  
   - 下载并安装Netron（[官网](https://netron.app/)）  
   - 直接双击 `glm5.onnx` 文件，或在命令行运行 `netron glm5.onnx`  
   - 你将看到一张节点图，每个操作（如MatMul、Add、LayerNorm）都以方块呈现，鼠标点击可查看参数信息。

### 以TensorKiko为例（更省事）
该工具无需转换格式，直接读取 `.safetensors` 文件：
```bash
pip install tensorkiko
tensorkiko --model_path /path/to/glm-5.2/model.safetensors
```
它会在浏览器中打开一个交互式树形图，你可以折叠/展开各层。

## 📊 三种方法对比

| 方法 | 优点 | 缺点 | 适用场景 |
|------|------|------|----------|
| **查看config.json** | 极快、无需运行代码 | 看不到具体子模块名称 | 快速获取宏观参数 |
| **代码打印** | 信息完整、可自定义输出 | 需要Python环境，输出冗长 | 需要详细结构或进行二次开发 |
| **可视化工具** | 直观、交互式 | 需额外转换或工具，大模型转换较慢 | 做演示、教学或深度探索 |

## 💡 针对GLM-5.2的特别提示

GLM-5.2除了常规的Transformer层外，还引入了：
- **IndexShare**：每4层稀疏注意力共享索引器，这在打印出的结构里可能会体现为多个注意力层共享某个权重，需要留意 `tie_weights` 等属性。
- **1M上下文**：其位置编码可能采用ALiBi或RoPE的变种，你可以在 `config.json` 中查看 `rope_theta` 或 `max_position_embeddings` 等相关字段。

建议将三种方法结合使用：先用 `config.json` 了解宏观，再用 `print(model)` 确认子模块，最后若需展示，用TensorKiko生成图。

## 解析每层结构作用

目前绝大多数大模型（如GPT、GLM等）都基于一种叫 **Transformer** 的架构。它就像一个高效的处理核心，其设计精髓可以概括为一个“**编码-解码**”的框架。

*   **编码器**：负责“理解”输入的文字，将其转化为机器能处理的“思维”表示。
*   **解码器**：负责“生成”输出，基于编码器的理解，逐字逐句地产生回答。

目前主流的大模型大多只使用**解码器**部分（如GPT系列），因为这种结构更擅长做文本生成任务。

无论架构如何变化，这个大模型“流水线”都由几个核心“工位”组成，每个工位各司其职。

### 🧱 第一站：输入嵌入与位置编码（原料准备与贴标签）

**1. 分词与嵌入 (Tokenization & Embedding)**
模型看不懂文字，首先要把句子切分成一个个“单词零件”（Token）。然后，每个Token会被转换成一个由一串数字组成的“**词向量**”。这个向量就像Token的“语义身份证”，其中数字代表了词语的各种特征，语义相近的词，其向量在数学空间中也比较接近。

**2. 位置编码 (Positional Encoding)**
Transformer不像人脑，它一次能看完所有文字，但**不区分先后顺序**。为了告诉模型“猫抓老鼠”和“老鼠抓猫”的区别，需要给每个位置的Token也生成一个独特的“位置向量”，并加到它的词向量上。

### 🧠 第二站：多头自注意力机制（全局信息统筹）

这是大模型最核心的“大脑”区域。它的任务是让每个词在生成自己的表示时，都能**纵观全局**，看看句子中其他词对它的影响。

其工作流程就像一个高效的会议：
1.  **生成Q、K、V**：每个Token都会生成三个向量：**查询(Query)**、**键(Key)** 和**值(Value)**。
2.  **计算关联度**：每个词的Q都会去和所有词的K计算一个“注意力分数”，这个分数决定了其他词对当前词的重要性。
3.  **加权汇总**：用计算出的注意力分数作为权重，对所有词的V进行加权求和，得到当前词新的、融合了上下文信息的表示。

而**“多头”** 机制，则是同时运行多个这样的“注意力会议”，每个“头”关注不同的方面（如语法、语义、指代等），最后将各“头”的结果综合起来，让模型的理解更全面。

### ⚙️ 第三站：前馈网络（深度思考与加工）

自注意力机制完成后，数据会进入前馈网络（FFN）。它就像一个**独立的、资深的思考者**，对每个Token的表示进行深度的独立加工。

*   **结构**：通常由两层全连接层组成。第一层会把数据的维度放大（比如从4096放大到11008），第二层再把维度压缩回原来大小。
*   **作用**：中间的非线性激活函数（如ReLU）为模型带来了**非线性变换**，让模型能够学习到更复杂的模式。研究表明，FFN层也是模型存储训练数据中“知识”的重要场所。

### 🏗️ 第四站：残差连接与层归一化（稳定与传承）

这两项技术是保证模型能够“越做越深”的关键。

*   **残差连接**：简单说就是“**抄近道**”。它让每一层的输入可以直接“跳过”本层处理，加到输出上。这有效防止了深层网络中的**梯度消失**问题，让信息能顺畅流动。
*   **层归一化**：它会对每一层神经元的输入进行“标准化”处理，让数据的分布保持稳定。这就像是给数据流做了一个“稳压”，能**加速模型收敛、提升训练稳定性**。

一个标准的Transformer层，就是按照 **“多头自注意力 → 残差连接 & 层归一化 → 前馈网络 → 残差连接 & 层归一化”** 的顺序，将上述组件串联起来。整个大模型就是由数十甚至上百个这样的层堆叠而成的。

### 💡 集大成者：以GLM-5.2为例

我们以你之前关注的GLM-5.2为例，来看看这些基础组件是如何被创新地应用在顶尖模型中的。

*   **稀疏MoE架构**：GLM-5.2总参数达**7440亿**，但实际激活的只有约**400亿**。这得益于其**混合专家（MoE）** 架构——每次计算时，模型只“激活”最擅长处理当前任务的部分“专家”网络，在不大幅增加计算成本的前提下，极大地扩展了模型的总知识容量。
*   **IndexShare机制**：为了处理**高达100万Token**的超长上下文，GLM-5.2提出了IndexShare创新。它在每四层稀疏注意力层之间**复用同一个索引器**，使得处理百万上下文时的计算量（FLOPs）**降低了2.9倍**。
*   **增强的推理能力**：GLM-5.2改进了其**MTP（多Token预测）层**，用于“投机解码”，将模型的推理接受长度**提升了20%**，显著加快了生成速度。

### 💎 总结

大语言模型的结构就像一个精密的协作系统：**嵌入层**将文字数字化，**位置编码**注入顺序信息，**多头注意力**负责捕获全局关联，**前馈网络**进行深度加工，而**残差连接**和**层归一化**则保证了整个系统的稳定运行。

像GLM-5.2这样的前沿模型，正是在这个经典框架之上，通过**稀疏MoE、IndexShare**等创新，在参数规模、上下文长度和推理效率上实现了新的突破。理解了这个基础“流水线”，你就能更好地把握几乎所有主流大模型的工作方式了。
