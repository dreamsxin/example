# Kali Linux

是一款为安全测试设计的操作系统，它有多种灵活的安装方式。这里根据不同的使用场景，为你梳理了四种方案。你可以根据下面的表格，快速选择最适合自己的那一种：

| 安装方式 | ⚙️ 推荐人群 | ⭐ 优点 | ❗ 注意事项 |
| :--- | :--- | :--- | :--- |
| **虚拟机 (VirtualBox/VMware)** | 所有用户，特别是初学者和开发者 | 最安全，与主系统完全隔离，可随意创建快照 | 性能受限于宿主机资源 |
| **双系统** | 追求极致性能、需要直接访问硬件的用户 | 硬件性能完全释放 | 分区操作复杂，有一定风险，需谨慎 |
| **USB Live (持久化)** | 需要便携、灵活环境的安全专家 | 即插即用，配置可保存 | 性能受限于U盘速度，配置更改后需手动保存 |
| **云服务部署** | 有远程测试需求的团队或个人 | 高可用性，随时随地访问 | 成本依据云服务商计费标准而定 |

### 🐧 Kali Linux 通用安装步骤

无论选择哪种安装方式，准备工作都大致相同：

1.  **系统要求**：确保你的设备至少有 2GB 内存（推荐 4GB 或以上）以及 20GB 的可用磁盘空间。
2.  **下载镜像**：从Kali Linux[官方网站](https://www.kali.org/get-kali/)下载所需的ISO镜像文件。
3.  **制作启动盘**：使用工具（如在Windows上用Rufus，或用跨平台的Balena Etcher）将下载的ISO镜像写入U盘，将其制作成可启动的安装介质。

完成上述准备后，你就可以根据选择的安装方式，参考以下步骤进行。

#### 💻 方案一：虚拟机安装
如果你对性能要求不高，但非常看重安全性与灵活性，虚拟机是你的最佳起点。这种方式能让Kali与你的主系统完全隔离，非常适合进行网络靶场、漏洞复现等安全实验，不会影响日常使用。

1.  **创建虚拟机**：打开VMware或VirtualBox，选择“创建新的虚拟机”。
2.  **加载镜像**：在设置中，选择“稍后安装操作系统”，然后在虚拟机光驱中加载你下载的Kali Linux ISO镜像文件。
3.  **配置资源**：为虚拟机分配至少 **2GB**（推荐4GB）内存和 **20GB** 的硬盘空间。
4.  **开始安装**：启动虚拟机，系统会从ISO镜像引导，进入Kali的安装界面，按照指引完成安装即可。

> **✨ 安装后的优化**
> 在虚拟机中，为了获得更流畅的体验（如自动调整分辨率、无缝鼠标切换），建议安装增强工具。VMware用户可运行 `sudo apt install open-vm-tools-desktop`，VirtualBox用户可运行 `sudo apt install virtualbox-guest-x11`。

#### 💻 方案二：双系统安装
虚拟机虽然方便，但当进行大规模密码破解或复杂渗透测试时，难免会感觉力不从心。如果你拥有独立的物理机，希望完全释放硬件的性能，双系统方案就是你的不二之选。

1.  **备份数据**：在操作分区前，务必备份所有重要数据。这项操作有风险，可能导致数据丢失。
2.  **准备空间**：在Windows的“磁盘管理”中，压缩出一个至少 **20GB** 的未分配空间，用于安装Kali Linux。
3.  **从U盘启动**：插入制作好的Kali启动U盘，重启电脑，并在BIOS/UEFI设置中将启动顺序调整为U盘优先。
4.  **开始安装**：进入Kali的图形化安装界面，选择**中文**语言。当安装程序询问分区方式时：
    *   **新手推荐**：选择**“向导 - 使用最大的连续空闲空间”**，安装程序会帮你自动完成分区，简单又安全。
    *   **经验用户**：选择**“手动”**，自行创建 swap（交换分区，大小与内存相当）和 ext4（根分区）分区。
5.  **安装引导程序**：安装Grub引导程序时，选择将它安装到整个硬盘的引导区。

这样，每次开机，你就能在Kali和主操作系统之间自由选择了。

#### 💻 方案三：U盘安装
Kali的官方镜像支持“Live USB”模式，这意味着系统可以直接在U盘上运行。这种方式适合临时使用，不会影响电脑硬盘上的任何数据。如果你希望U盘上的系统能保存配置和文件，只需在制作启动盘时额外配置一个“持久化存储”分区。

#### 💻 方案四：云服务器安装
如果你需要一台24小时在线的Kali环境，用于远程渗透测试或持续监控，那么部署在云端是最高效的选择。目前，主流的云服务商如AWS、Azure、DigitalOcean等都提供预配置好的官方Kali Linux镜像。

*   **AWS部署**：以AWS为例，你只需登录AWS管理控制台，进入EC2服务，在“社区AMI”中搜索“Kali Linux”选择合适的版本，然后按向导启动实例即可。
*   **其他平台**：对于不支持预装镜像的国内云服务商，你也可以通过上传自定义镜像等方式进行部署。

### 🚨 故障排除与常见问题

1.  **启动失败**：可能是ISO文件损坏或启动盘制作失败。建议重新下载ISO，用Rufus或Etcher这类可靠的工具重新制作启动盘。对于物理机，也请检查BIOS是否已正确关闭 **UEFI/Secure Boot** 选项。
2.  **分区问题**：新手可以使用安装程序提供的“向导”模式，它会自动利用硬盘的空闲空间进行分区，避免手动操作失误。
3.  **硬件兼容性**：比如显卡驱动问题或触摸板失灵。可以先尝试更新系统和内核。对于触摸板，安装 `xserver-xorg-input-libinput` 驱动可能解决大部分问题。如果仍有问题，可以参考Kali官方文档的图形问题故障排除。新手也可以收藏一些汇总常见报错的对照表，方便快速排查。
4.  **Root密码**：Kali 2020版本之后，默认不启用root用户。在安装过程中会提示你创建一个普通用户，安装完成后使用这个普通用户登录，并通过 `sudo` 执行管理员命令。
