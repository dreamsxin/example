# Nmap

在 Kali Linux 中进行端口扫描，最核心、最常用的工具就是 **Nmap**。它就像一个多功能的“网络雷达”，不仅能发现目标主机，还能精确识别端口开放状态和服务信息，是安全分析人员的必备工具。

为了让你快速上手，我这里先梳理一下不同扫描目标的命令速查：

| 扫描目标 | 命令示例 | 说明 |
| :--- | :--- | :--- |
| **单个IP** | `nmap 192.168.1.100` | 最基础的用法，扫描该IP的1000个常见端口 |
| **整个子网** | `nmap 192.168.1.0/24` | 扫描 `/24` 网段内的所有主机，适用于局域网发现 |
| **多个IP** | `nmap 192.168.1.1 192.168.1.10` | 扫描列表中指定的IP地址 |
| **IP范围** | `nmap 192.168.1.1-10` | 扫描从 `.1` 到 `.10` 的IP地址 |

### 🎯 Nmap 核心功能详解

Nmap之所以强大，是因为它提供了丰富的扫描参数来控制扫描的深度和方式。

#### 端口扫描技术
选择合适的扫描技术是成功的第一步：

*   **SYN扫描（-sS）**：默认的扫描方式，它发送SYN包，若收到SYN/ACK则回复RST包断开连接，因此不会在目标系统日志中留下完整连接记录，兼具隐蔽和高效的优点。
*   **TCP Connect扫描（-sT）**：调用系统网络API完成TCP三次握手，连接建立成功即代表端口开放，这种方式最稳定、不易出错，但最容易被防火墙等安全设备记录。
*   **UDP扫描（-sU）**：DNS(53)、SNMP(161)等UDP服务扫描，由于UDP无连接的特性，通常需要构造特定请求并等待回应，整体速度会慢很多。

#### 扫描深度调整
除了了解端口是否开放，Nmap可以帮你获取更详细的信息：

*   **服务/版本探测（-sV）**：深入分析开放端口，识别其运行的服务名称和具体版本。
*   **操作系统探测（-O）**：通过分析目标主机TCP/IP协议栈的细微特征，推断其操作系统类型和版本。
*   **脚本扫描（-sC 或 --script=脚本名）**：Nmap内置了强大的NSE脚本引擎。`-sC`参数可以运行默认的安全审计脚本，更可以指定脚本进行漏洞检测等高级操作。
*   **全面综合扫描（-A）**：这是一个进阶参数，相当于同时启用了`-O`（操作系统检测）、`-sV`（版本检测）、`-sC`（默认脚本扫描）和路由追踪等多项功能，为你提供最全面的信息。

#### 端口选择与性能优化
*   **指定端口范围（-p）**：
    *   扫描单个端口：`nmap -p 80 <目标>`
    *   扫描多个指定端口：`nmap -p 80,443,8080 <目标>`
    *   扫描端口范围：`nmap -p 1-1000 <目标>`
    *   扫描全部65535个端口：`nmap -p- <目标>`
*   **常见端口快速扫描（-F）**：有时我们只想快速确认主机是否开放了如21、22、80等最常用的端口。此时可以使用 `-F` 参数，它只会扫描Nmap内置的约100个最有可能开放的端口，极大地缩短扫描时间。
*   **控制扫描速度（-T）**：通过`-T0`到`-T5`的时序模板来调整扫描速度。默认是`-T3`。`-T4`会稍快一些，而`-T5`则达到最快速度。通常建议使用`-T4`。
*   **主机发现**：Nmap通过Ping扫描（`-sn`）或ARP扫描（`-PR`）等确定目标是否在线，可有效提高扫描效率。
*   **输出保存（-oN, -oX, -oG）**：`-oN`保存为普通文本，`-oX`保存为XML便于程序化处理，`-oG`保存为“grepable”格式方便grep过滤。

### 🚀 进阶技巧与工具

#### 1. 防火墙规避
在实际环境中，目标主机或网络设备很可能部署了防火墙。Nmap本身具备多种绕过技巧：

*   **数据包分片 (`-f`)**：将探测数据包分成更小的碎片，部分配置较低的防火墙可能无法正确重组，从而让其绕过。
*   **诱饵扫描 (`-D`)**：制造多个虚假的源IP地址（“诱饵”）和你一起发起扫描，使得目标看到的扫描来自多个IP，从而隐藏你的真实身份。例如：`nmap -D RND:10,192.168.1.101,ME 192.168.1.1`。
*   **指定源端口 (`--source-port`)**：一些防火墙配置会信任来自特定端口（如`80`）的流量。你可以将扫描源端口设置为这些常用端口来尝试绕过。
*   **修改TTL (`--ttl`)**：可以修改你发送的IP数据包的`time to live`值，以便通过一些基于TTL值进行过滤的防火墙。

#### 2. 高速端口扫描：Masscan
当需要对一个大型网络进行快速探测时，Nmap的速度可能稍显不足。这时，Masscan是你的最佳选择。
*   专为速度设计，能在极短的时间内扫描整个互联网。
*   它的核心是异步、无状态的工作方式，它可以和Nmap无缝配合，通常先用Masscan快速发现开放端口，再用Nmap对这些端口进行详细分析。
*   **一个典型的Masscan命令示例**：
    ```bash
    # 扫描192.168.1.0/24网段的80, 443, 8080端口，速率10000包/秒
    sudo masscan 192.168.1.0/24 -p80,443,8080 --rate=10000
    ```
*   **💡 实用技巧**：在Kali Linux中使用Masscan时，为了避免系统内核发起的RST包干扰扫描结果，添加 `--source-port 61000` 参数并结合 `iptables` 规则会是个高效的做法。

#### 3. 更多其他工具
除了Nmap和Masscan，Kali Linux中还内置了其他一些专业工具，各有侧重：
*   **Zmap**：与Masscan类似，也是专为高速、大规模网络扫描而设计的工具。
*   **Unicornscan**：在设计上兼顾了速度和一定深度，能够进行服务识别。
*   **Netcat**：一个被称为“瑞士军刀”的简单网络工具，通过脚本可以执行基础的端口扫描。
*   **Zenmap**：Nmap的官方图形化界面（GUI），如果你不习惯使用命令行，它是非常适合新手的入门选择。

### ⚠️ 法律与道德提醒

端口扫描是一项强大的网络探测技术，请务必在**获得明确授权**的情况下，对自己拥有所有权的网络或设备进行扫描，或用于合法的安全审计工作。未经授权的端口扫描在部分国家或地区可能触犯相关法律法规，请严格遵守网络安全法律。
