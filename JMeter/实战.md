## Jmeter 使用说明

### 基础命令参数介绍

Jmeter 提供了丰富的命令行参数，以下是一些常用的参数及其功能：

- -n 或 --nongui：以非图形界面模式运行 Jmeter，适用于服务器环境或自动化测试场景。
- -t 或 --testfile：指定要运行的测试脚本文件（.jmx 格式），支持相对路径或绝对路径。
- -l 或 --logfile：将测试结果保存到指定文件（如 .jtl 或 .csv 格式），便于后续分析。
- -e 或 --reportDirectory：在测试结束后生成 HTML 格式的报告，需配合 -o 参数使用。
- -o 或 --outputDirectory：指定报告输出的目录路径。

### 示例命令解析

针对您提供的命令 `.\jmeter.bat -n -t .\xxxx.jmx -l file.csv -e -o dir`，其含义如下：

`.\jmeter.bat`：调用 Jmeter 的 Windows 启动脚本。
- -n：以非图形界面模式运行。
- -t .\google_risk_test.jmx：执行当前目录下的 google_risk_test.jmx 测试脚本。
- -l test23.csv：将测试结果保存为 test23.csv 文件。
- -e -o test23：测试结束后生成 HTML 报告，并输出到 test23 目录。
其他常用参数
- -r 或 --runremote：在非 GUI 模式下启动 jmeter.properties 中配置的所有远程服务器。
- -R 或 --remotestart：指定要启动的远程服务器列表，覆盖 jmeter.properties 中的配置。
- -J 或 --jmeterproperty：设置 Jmeter 的本地属性，例如 -J num_threads=100 调整线程数。
- -D 或 --systemproperty：设置 Java 系统属性，如 -Djava.net.preferIPv4Stack=true。

### 注意事项

脚本路径和结果文件路径需使用双引号包裹（如 "C:\test\script.jmx"），避免空格或特殊字符导致错误。
生成报告前需确保测试结果文件（如 .jtl）已正确生成且格式有效。
远程服务器测试需提前配置 `jmeter.properties` 中的 `remote_hosts` 参数。

如需进一步了解参数细节，可通过 `jmeter -h` 查看帮助信息。
