# Canvas 2D 等距游戏框架设计文档

本文档提供一套基于原生 Canvas 2D API 的 2.5D 等距/二测游戏完整架构设计。该框架采用模块化设计，支持模拟经营、RPG 等常见玩法，并手动处理投影变换、深度排序和精灵渲染，无需任何外部图形库。  
**新增特性**：初始化时可选择投影类型（等距/二测）和视角角度（默认45°），灵活适应不同视觉风格。

---

## 一、 设计目标

- 利用 Canvas 2D 实现固定等距/二测视角的 2.5D 游戏世界。
- 支持投影类型（isometric / dimetric）和视角角度（view angle）配置。
- 提供清晰的模块划分，支持高内聚低耦合的开发模式。
- 手动处理深度排序，确保角色与建筑的正确遮挡关系。
- 支持鼠标/触摸交互、网格寻路、动态加载。
- 具备良好的调试能力和性能优化空间。

## 三、 核心模块详解

### 1. Canvas 渲染核心

#### 1.1 Projection（投影管理器）

**职责**：根据投影类型和视角角度，管理世界坐标到屏幕坐标的投影变换，并提供逆变换方法。

**关键数据结构**：
```typescript
interface ProjectionConfig {
    type: 'isometric' | 'dimetric';   // 投影类型
    viewAngle: number;                 // 水平旋转角度（度），默认 45
    tiltAngle?: number;                // 俯仰角（度），仅 dimetric 使用，默认 30
    tileScale?: number;                // 缩放因子，默认 1
}

class Projection {
    public type: string;
    public viewAngleRad: number;
    public tiltAngleRad: number;
    public cosView: number;
    public sinView: number;
    public sinTilt: number;
    public cosTilt: number;

    constructor(config: ProjectionConfig);
    public worldToScreen(x: number, z: number, y: number): { sx: number, sy: number };
    public screenToWorld(sx: number, sy: number, y: number): { x: number, z: number };
}
```

**实现要点**：
- **等距投影（isometric）**：固定 `tiltAngle` 为 30°（或根据 `viewAngle` 计算），使用标准等距公式：
  ```typescript
  worldToScreen(x, z, y) {
      const screenX = (x - z) * this.cosView * this.tileScale;
      const screenY = (x + z) * this.sinView * this.tileScale - y;
      return { sx: screenX, sy: screenY };
  }
  ```
  其中 `viewAngle` 通常为 45°（`cos=sin=√2/2`），此时公式简化为常见菱形投影。
- **二测投影（dimetric）**：允许独立控制水平旋转和俯仰角：
  ```typescript
  worldToScreen(x, z, y) {
      const screenX = (x * this.cosView - z * this.sinView) * this.tileScale;
      const screenY = (x * this.sinView + z * this.cosView) * this.sinTilt - y * this.cosTilt;
      return { sx: screenX, sy: screenY };
  }
  ```
- **逆变换**：对于等距投影，可解析求解；对于二测投影，假设 y=0 并解线性方程组。具体实现：
  ```typescript
  screenToWorld(sx, sy, y = 0) {
      if (this.type === 'isometric') {
          const x = (sx / this.cosView + sy / this.sinView) / 2 / this.tileScale;
          const z = (sy / this.sinView - sx / this.cosView) / 2 / this.tileScale;
          return { x, z };
      } else { // dimetric
          // 解方程组: [cos, -sin; sin, cos] * [x; z] = [sx / scale; sy / sinTilt + y*cosTilt/sinTilt]
          const a = this.cosView, b = -this.sinView, c = this.sinView, d = this.cosView;
          const det = a * d - b * c;
          const tx = sx / this.tileScale;
          const ty = sy / this.sinTilt + y * this.cosTilt / this.sinTilt;
          const x = (d * tx - b * ty) / det;
          const z = (-c * tx + a * ty) / det;
          return { x, z };
      }
  }
  ```

#### 1.2 CanvasRenderer（渲染器）

**职责**：封装 Canvas 元素和 2D 上下文，管理每帧待渲染对象列表，执行深度排序和最终绘制。  
**修改**：引入 `Projection` 实例，所有坐标转换委托给它。

**关键数据结构**：
```typescript
class CanvasRenderer {
    public canvas: HTMLCanvasElement;
    public ctx: CanvasRenderingContext2D;
    public camera: IsoCamera;
    public projection: Projection;          // 新增
    private renderItems: RenderItem[];

    constructor(width: number, height: number, projection: Projection);
    public addRenderItem(item: RenderItem): void;
    public addRenderItems(items: RenderItem[]): void;
    public clearRenderItems(): void;
    public render(): void;
    public worldToScreen(x: number, z: number, y?: number): { sx: number, sy: number };
}
```

**实现要点**：
- `worldToScreen` 直接调用 `projection.worldToScreen`，然后应用相机偏移和缩放。
- 渲染前应用相机变换时，需要将投影后的屏幕坐标进一步变换。

#### 1.3 IsoCamera（等距相机）

**职责**：模拟相机位置、缩放和滚动。**注意**：相机偏移量针对的是投影后的屏幕空间，而不是原始世界空间。

**关键数据结构**：
```typescript
class IsoCamera {
    public offsetX: number;      // 屏幕空间偏移（像素）
    public offsetY: number;
    public scale: number;
    constructor(offsetX: number, offsetY: number, scale: number);
    public setPosition(worldX: number, worldZ: number, projection: Projection): void;
    public zoom(delta: number): void;
    public applyTransform(ctx: CanvasRenderingContext2D): void;
}
```

**实现要点**：
- `setPosition` 根据给定世界坐标计算屏幕偏移，使该点位于屏幕中心。
- 相机变换与投影无关，只需在渲染前对上下文进行平移和缩放即可。

#### 1.4 RenderItem（渲染项接口）

**职责**：所有可渲染对象必须实现此接口，用于排序和绘制。

```typescript
interface RenderItem {
    depth: number;          // 深度值，数值越小越先绘制（远）
    draw(ctx: CanvasRenderingContext2D): void;
    update?(delta: number): void; // 可选，动态更新逻辑
}
```

### 3. 游戏世界管理模块

#### 3.1 GridSystem（网格系统）

**职责**：管理地图数据、坐标转换、地面图块渲染。  
**修改**：网格坐标与世界坐标的转换仍然基于菱形逻辑，但实际世界坐标会被投影到屏幕。地面图块生成时，需要知道投影参数以计算屏幕位置。

**关键数据结构**：
```typescript
class GridSystem {
    private width: number;
    private height: number;
    private tiles: TileData[][];
    private tileW: number;           // 格子宽度（世界单位）
    private tileH: number;           // 格子深度（世界单位）
    private projection: Projection;   // 引用投影实例
}
```

**主要方法**：
- `gridToWorld(col, row)`：与之前相同。
- `worldToGrid(x, z)`：与之前相同。
- `buildGroundRenderItems(camera: IsoCamera): RenderItem[]`：生成地面图块渲染项，每个图块的屏幕位置通过 `projection.worldToScreen` 计算，深度值取屏幕 Y 坐标。

**实现要点**：
- 地面图块的绘制：使用 `drawImage` 绘制菱形图块，位置由投影后的屏幕坐标确定。
- 性能优化：可预先将所有地面图块绘制到离屏 canvas，每帧只绘制该 canvas。

#### 3.2 EntityManager（实体管理器）

**职责**：管理所有游戏实体。  
**修改**：实体深度值计算使用投影后的屏幕 Y 坐标。

**关键数据结构**：
```typescript
abstract class Entity implements RenderItem {
    // ... 原有属性
    public depth: number;            // 每帧更新
    public getScreenPosition(projection: Projection, camera: IsoCamera): { sx: number, sy: number };
}
```

**实现要点**：
- `getScreenPosition` 调用 `projection.worldToScreen` 后应用相机偏移和缩放。
- 深度值：`depth = projection.worldToScreen(x, z, y).sy + camera.offsetY`（或直接取屏幕 Y 坐标）。

#### 3.3 PathFinder（寻路系统）

**职责**：基于网格地图，提供 A* 寻路算法，支持地形代价和动态障碍物。

**关键数据结构**：
```typescript
interface PathConfig {
    allowDiagonal: boolean;     // 是否允许对角线移动（默认 false）
    terrainCost: Map<string, number>; // 地形类型 -> 移动代价
}
```

**主要方法**：
- `findPath(start, end, gridSystem, config?)`：返回路径点数组（不含起点），若不可达返回空数组。

**实现要点**：
- 使用优先队列（二叉堆）管理 open 集。
- 启发函数使用曼哈顿距离。
- 动态查询 `gridSystem.isWalkable` 考虑实体占用。

## 四、 模块间协作示例（投影相关）

以下以“玩家点击移动”为例，展示投影模块如何参与交互：

1. **游戏初始化**：根据配置创建 `Projection` 实例，传入 `CanvasRenderer`、`GridSystem`、`InputManager` 等。
2. **InputManager** 捕获 `click`，获取屏幕坐标 `(sx, sy)`，先应用相机逆变换得到未缩放偏移的投影坐标，再调用 `projection.screenToWorld` 获得世界坐标 `(x, z)`。
3. 世界坐标通过 `GridSystem.worldToGrid` 转为网格坐标 `(col, row)`，触发 `tileClick` 事件。
4. 后续移动流程与之前相同，实体移动时更新世界坐标，渲染时通过 `projection.worldToScreen` 计算屏幕位置。

## 五、 坐标系统与投影公式

### 5.1 世界坐标系
- X 轴向右，Z 轴向下（与屏幕平面平行），Y 轴向上。
- 网格坐标 `(col, row)` 转世界坐标（地面，y=0）：
  ```
  x = (col - row) * (tileW / 2)
  z = (col + row) * (tileH / 2)
  ```

### 5.2 投影公式

#### 等距投影（isometric）
```
screenX = (x - z) * cos(viewAngle) * tileScale
screenY = (x + z) * sin(viewAngle) * tileScale - y
```
当 `viewAngle = 45°` 时，`cos=sin≈0.707`，公式简化为常见形式。

#### 二测投影（dimetric）
```
screenX = (x * cos(viewAngle) - z * sin(viewAngle)) * tileScale
screenY = (x * sin(viewAngle) + z * cos(viewAngle)) * sin(tiltAngle) - y * cos(tiltAngle)
```
其中 `tiltAngle` 为俯仰角（默认 30°），控制垂直方向压缩。

### 5.3 逆投影（屏幕 → 世界，假设 y=0）
- **等距**：可解析求解
  ```
  x = (screenX / cos(viewAngle) + screenY / sin(viewAngle)) / 2 / tileScale
  z = (screenY / sin(viewAngle) - screenX / cos(viewAngle)) / 2 / tileScale
  ```
- **二测**：解线性方程组
  ```
  [ cos(viewAngle)  -sin(viewAngle) ] [x] = [ screenX / tileScale ]
  [ sin(viewAngle)   cos(viewAngle) ] [z]   [ screenY / (sin(tiltAngle)) ]   (忽略 y 项)
  ```
  通过矩阵求逆得到 `(x, z)`。

## 六、 深度排序与遮挡处理

深度值基于投影后的屏幕 Y 坐标计算，因此投影类型和角度会影响物体的相对深度顺序。例如，在二测投影中，俯仰角的变化会改变物体在屏幕上的垂直位置，从而影响遮挡关系。框架的排序机制仍然有效，只要所有物体使用相同的投影参数即可。

- **深度值定义**：物体的深度 = 它在屏幕上的底部 Y 坐标（考虑高度偏移）。公式：`depth = worldToScreen(x, z, y).sy`。
- **排序规则**：深度值小的物体先绘制（远处），深度值大的后绘制（近处），自然实现近处遮挡远处。
- **实体高度影响**：角色或建筑的高度增加，其深度值也增加，因此会绘制在地面之上，且高建筑会遮挡后面的低建筑。
- **半透明物体**：若需要透明效果，在 `draw` 中设置 `ctx.globalAlpha`。但半透明物体排序需额外处理，建议将它们独立成一层，最后绘制并启用混合模式。

## 七、 性能优化策略

1. **地面图块合并**：将静态地面图块预渲染到离屏 canvas，每帧只绘制一张大图。但由于等距图块需要精确位置，可以先将每个格子的图块绘制到离屏 canvas 上，形成完整的背景图，每帧只需复制该背景图。
2. **视域剔除**：只生成相机视野内的地面渲染项。根据相机位置和缩放计算可见的网格范围，避免渲染屏幕外的图块。
3. **精灵图集**：将所有角色、建筑纹理合并到一张大图中，使用 `drawImage` 的裁剪参数绘制，减少纹理切换开销。
4. **对象池**：对频繁创建销毁的特效、子弹等使用对象池，减少 GC 压力。
5. **分帧更新**：对于不需要每帧更新的逻辑（如 AI 决策），可采用间隔更新策略。
6. **降帧渲染**：如果游戏逻辑简单，可以限制更新频率（如固定 30fps），但动画仍需平滑。

## 八、 游戏循环与事件驱动

- **主循环**：使用 `requestAnimationFrame` 驱动。
- **每帧流程**：
  1. 计算时间差 `delta`（限制最大值防止跳跃过大）。
  2. 调用 `SceneManager.getCurrentScene().update(delta)` 更新场景内所有模块。
  3. 更新相机（如有平滑跟随）。
  4. 清空渲染队列，收集静态地面渲染项（或直接绘制离屏背景）。
  5. 调用 `EntityManager.getRenderItems()` 获取动态物体渲染项。
  6. 调用 `CanvasRenderer.render()` 进行排序和绘制。
  7. 调用 `UIManager` 更新 HTML UI（如有）。
  8. 调用 `DebugSystem` 绘制调试信息（如果启用）。
- **事件驱动**：所有模块间的非直接调用通过 `EventBus` 完成，保持松耦合。

# 输入与交互模块详细设计

## 一、 InputManager（输入管理器）

### 1. 职责
- 统一捕获鼠标、键盘、触摸等原始输入事件。
- 将屏幕坐标转换为游戏世界坐标（通过投影和相机）。
- 将原始输入转换为高层游戏事件（如格子点击、实体点击、按键命令），并通过 `EventBus` 发布。
- 管理输入状态（如按键持续按下、拖拽、手势）。
- 提供输入屏蔽机制（当 UI 模态窗口打开时暂停世界输入）。

### 2. 关键数据结构


### 3. 主要方法

#### 初始化与销毁
- `init(canvas: HTMLCanvasElement, camera: IsoCamera, projection: Projection, eventBus: EventBus): void`  
  绑定 DOM 事件，初始化内部状态。
- `destroy(): void`  
  解绑事件，清理资源。

#### 事件处理
- `private onMouseClick(event: MouseEvent): void`

- `getWorldPositionFromScreen(screenX: number, screenY: number): { x: number, z: number, y: number }`  
  将屏幕坐标转换为世界坐标。步骤：
  1. 将屏幕坐标转为相对于 canvas 的坐标（考虑 CSS 缩放）。
  2. 反向应用相机变换（`camera` 的平移和缩放）。
  3. 调用 `projection.screenToWorld` 得到世界坐标 `(x, z)`。
  4. 可通过射线检测地面获得 Y 值（如果需要精确高度，可用 `GridSystem.getGroundHeight`）。

#### 辅助功能
- `setBlocking(block: boolean): void`  
  设置是否屏蔽世界输入（由 UIManager 调用）。
- `isKeyPressed(key: string): boolean`  
  查询按键状态。

### 4. 实现要点

#### 4.1 屏幕坐标转换步骤
由于相机可能缩放和平移，需要反向计算：
```typescript
function getWorldFromScreen(screenX, screenY) {
    // 1. 计算相对于 canvas 的原始像素坐标
    const rect = canvas.getBoundingClientRect();
    const canvasX = (screenX - rect.left) * (canvas.width / rect.width);
    const canvasY = (screenY - rect.top) * (canvas.height / rect.height);
    
    // 2. 反向应用相机缩放和平移
    // 相机变换公式：screen = (world - offset) * scale + center
    // 因此 world = (screen - center) / scale + offset
    const centerX = canvas.width / 2;
    const centerY = canvas.height / 2;
    const worldX = (canvasX - centerX) / camera.scale + camera.offsetX;
    const worldY = (canvasY - centerY) / camera.scale + camera.offsetY;
    
    // 3. 通过投影逆变换得到世界坐标 (x, z)
    // 注意：worldX, worldY 是投影后的屏幕坐标（未应用相机偏移的部分）
    const { x, z } = projection.screenToWorld(worldX, worldY, 0);
    return { x, z };
}
```

#### 4.2 实体点击检测
- 通过 `EntityManager.getEntities()` 遍历所有实体。
- 获取实体的屏幕包围盒（根据世界坐标 + 精灵尺寸计算）。
- 判断鼠标点是否在包围盒内。
- 按深度排序，取最上层（深度最大）的实体作为点击目标。

#### 4.3 拖拽与滚动
- 监听 `mousedown` + `mousemove` 实现相机拖拽（平移）。
- 监听 `wheel` 事件实现相机缩放。
- 拖拽时设置 `dragThreshold` 避免误触发点击。

#### 4.4 键盘映射
- 将常用按键（WASD、箭头键）映射为移动命令，通过 `EventBus` 发布 `playerMove` 事件。
- 支持组合键（Ctrl+Z 等）用于撤销等高级功能。

#### 4.5 触摸支持
- 将触摸点模拟为鼠标事件，支持单点拖拽和多点缩放。
- 使用 `preventDefault` 避免页面滚动。

### 5. 事件发布列表

| 事件名 | 数据 | 描述 |
|--------|------|------|
| `tileClick` | `{ col, row }` | 点击了某个格子（无实体时） |
| `entityClick` | `{ entityId }` | 点击了某个实体 |
| `mouseMove` | `{ screenX, screenY, worldX, worldZ }` | 鼠标移动，用于 UI 高亮 |
| `keyDown` | `{ key, code }` | 按键按下 |
| `keyUp` | `{ key, code }` | 按键释放 |
| `dragStart` | `{ startX, startY }` | 开始拖拽相机 |
| `dragMove` | `{ deltaX, deltaY }` | 拖拽中 |
| `dragEnd` | `{ }` | 拖拽结束 |
| `zoom` | `{ delta }` | 缩放操作 |

### 6. 模块协作
- **UIManager**：通过 `setBlocking` 通知 InputManager 是否屏蔽世界输入（如打开菜单时）。
- **Camera**：获取相机参数用于坐标转换。
- **Projection**：执行投影逆变换。
- **EntityManager**：用于实体点击检测。

## 二、 UIManager（UI管理器）

### 1. 职责
- 管理所有 2D UI 组件（使用 HTML/CSS 叠加层，或独立 Canvas 层）。
- 负责 UI 的创建、显示、隐藏和销毁。
- 处理 UI 与游戏世界的交互（如按钮点击、对话框确认），通过 `EventBus` 与游戏逻辑通信。
- 控制输入屏蔽：当 UI 模态窗口打开时，设置 `InputManager.blocking = true`，防止点击穿透。
- 提供标准 UI 组件（对话框、血条、工具栏、小地图等）的快速构建方法。

### 2. 关键数据结构

```typescript
    private eventBus: EventBus;
    private inputManager: InputManager;
    private debugEnabled: boolean;            // 调试模式，显示额外 UI 信息
}
```

### 3. 主要方法

#### 组件管理
- `addComponent(component: UIComponent): void`  
  添加一个 UI 组件，将其元素附加到容器中。如果组件是模态的，将其加入模态栈并屏蔽输入。
- `removeComponent(id: string): void`  
  移除组件，清理 DOM 元素，从模态栈中删除，并更新输入屏蔽状态。
- `showComponent(id: string): void`  
  显示组件，如果是模态组件则更新模态栈。
- `hideComponent(id: string): void`  
  隐藏组件，如果是模态组件则更新模态栈。

- `createDialog(title: string, content: string, onConfirm: () => void, onCancel?: () => void): UIComponent`

  显示临时浮动提示消息。

- `private updateInputBlocking(): void`

### 4. 实现要点

#### 4.1 UI 容器布局
- 使用绝对定位的 `<div>` 覆盖整个 canvas，设置 `pointer-events: none`，内部 UI 组件可设置 `pointer-events: auto` 来接收点击。
- 通过 CSS 控制组件位置、样式、动画。

#### 4.2 模态窗口
- 模态窗口会生成一个半透明遮罩层，并阻止世界输入。
- 窗口关闭时，从模态栈弹出，若栈为空则恢复输入。

#### 4.3 事件通信
- UI 组件中的按钮点击应通过 `eventBus.emit('uiAction', { componentId, action })` 触发，由游戏逻辑模块监听处理。
- 游戏逻辑模块可通过 `eventBus.emit('updateUI', { type, data })` 更新 UI。

#### 4.4 性能优化
- 避免频繁 DOM 操作，使用 `requestAnimationFrame` 批量更新。
- 对于频繁更新的 UI（如血条），可考虑使用 Canvas 绘制（在 `DebugSystem` 中实现），减少 DOM 重排。

#### 4.5 响应式布局
- 监听窗口 resize，重新计算 UI 组件位置（相对 canvas 大小）。

## 三、 DebugSystem（调试系统）

### 1. 职责
- 提供开发时的可视化调试工具，帮助定位问题。

- `init(canvasRenderer, gridSystem, entityManager, inputManager, eventBus): void`  
  注入依赖，监听快捷键事件。
- `toggle(): void`  
  切换调试模式开关。
- `setConfig(config: Partial<DebugConfig>): void`  
  单独设置调试选项。

#### 绘制方法
- `draw(ctx: CanvasRenderingContext2D): void`  
  在 `CanvasRenderer` 渲染结束后调用，绘制所有调试信息。
- `private drawGrid(ctx): void`  
  遍历所有格子，根据投影计算屏幕坐标，绘制菱形网格线。
- `private drawCoordinates(ctx): void`  
  在每个格子中心绘制坐标文本。
- `private drawBoundingBoxes(ctx): void`  
  获取所有实体的屏幕包围盒，绘制矩形。
- `private drawMouseInfo(ctx): void`  
  在鼠标位置附近显示当前格子的坐标、地形类型、实体等信息。
- `private drawFPS(ctx): void`  
  在屏幕角落显示 FPS 和帧时间。
- `private drawStats(ctx): void`  
  显示 draw calls、实体数量等。

#### 性能统计
- `private updateStats(): void`  
  每帧计算 FPS（基于 `requestAnimationFrame` 的时间差）。
- `private collectDrawCalls(): number`  
  可从 `CanvasRenderer` 获取本次渲染的 `renderItems` 数量作为近似 draw calls。

### 4. 实现要点

#### 4.1 调试信息的绘制时机
- 在 `CanvasRenderer.render()` 中，完成正常渲染后，如果 `debugSystem.enabled` 为 true，调用 `debugSystem.draw(ctx)`。
- 或者使用独立的 canvas 层叠加，但通常在同一 canvas 上绘制更简单（注意清除上一帧的调试信息）。

#### 4.2 网格线绘制
- 根据 `gridSystem` 获取所有格子，计算每个格子的四个顶点屏幕坐标，使用 `ctx.beginPath()` 绘制线段。
- 为了性能，可以只绘制可见区域内的格子，但调试模式下通常全地图绘制也不会有太大开销。

#### 4.3 坐标文本
- 使用 `ctx.fillText` 绘制坐标，注意文本方向不要被投影扭曲。
- 可设置 `ctx.font` 和 `ctx.textAlign` 居中。

#### 4.4 包围盒绘制
- 实体的包围盒大小基于其精灵尺寸（可预定义）和高度，计算屏幕坐标下的矩形。
- 使用 `ctx.strokeRect` 绘制矩形框，可选半透明填充。

#### 4.5 鼠标信息
- 监听 `mouseMove` 事件，获取当前鼠标下的网格坐标和实体信息，显示在工具提示中。

#### 4.6 性能统计
- FPS 计算：记录上一帧时间，计算间隔的倒数。
- draw calls：对于 Canvas 2D，draw calls 约等于 `drawImage` 和 `fill` 等调用的次数，可从 `CanvasRenderer` 中获取 `renderItems.length` 作为近似。
- 实体数量：从 `entityManager` 获取。

#### 4.7 快捷键控制
- 监听 `keydown` 事件（如 `` ` `` 或 `~`），调用 `toggle()`。
- 可支持组合键如 `Ctrl+Shift+D` 切换详细模式。

### 5. 模块协作
- **CanvasRenderer**：提供 `ctx` 和渲染流程入口。
- **GridSystem**：提供格子数据用于绘制网格和坐标。
- **EntityManager**：提供实体列表用于包围盒。
- **InputManager**：提供鼠标位置信息。
- **EventBus**：监听调试模式切换事件（`toggleDebug`）。

### 6. 事件与配置
- `eventBus.on('toggleDebug', () => debugSystem.toggle())`：外部可通过事件开启调试。
- 调试配置可通过本地存储保存，方便下次启动恢复。

# Game World Manager 模块详细设计

## 一、 GridSystem（网格系统）

### 1. 职责
- 管理游戏世界的二维网格数据，包括地形类型、高度、可行走性、实体占用等。
- 提供网格坐标与平面世界坐标（XZ 平面）的双向转换。
- 生成地面图块的渲染项（RenderItem），供 CanvasRenderer 绘制。
- 管理地面高度信息，影响实体 Y 坐标。
- 维护实体占用映射，支持快速查询指定格子上的实体。
- 与 EntityManager 协作更新占用状态。

### 2. 关键数据结构

```typescript
// 单个格子数据
interface TileData {
    type: string;               // 地形类型标识，如 'grass', 'water', 'road'
    height: number;             // 海拔高度（世界单位），影响 Y 坐标
    walkable: boolean;          // 是否可行走
    entity: Entity | null;      // 当前占据此格子的实体（若有）
    // 可选：其他属性，如建造标记、资源量等
}

// 地图配置参数
interface MapConfig {
    width: number;              // 网格列数（col 方向）
    height: number;             // 网格行数（row 方向）
    tileW: number;              // 格子宽度（世界单位）
    tileH: number;              // 格子深度（世界单位）
}

// 地面渲染项（用于绘制菱形图块）
class TileRenderItem implements RenderItem {
    public depth: number;
    public col: number;
    public row: number;
    public screenX: number;
    public screenY: number;
    public image: HTMLImageElement;    // 根据地形类型从资源管理器获取
    public frameRect?: { x: number, y: number, w: number, h: number }; // 图集裁剪

    constructor(depth: number, col: number, row: number, screenX: number, screenY: number, image: HTMLImageElement);
    public draw(ctx: CanvasRenderingContext2D): void;
}
```

### 3. 主要方法

#### 初始化与数据管理
- `init(config: MapConfig, tileData: TileData[][]): void`  
  根据地图配置和初始数据构建网格。
- `getTile(col: number, row: number): TileData | null`  
  获取指定格子的数据（边界检查）。
- `setTileType(col: number, row: number, type: string, walkable: boolean): void`  
  动态修改地形类型（如建造/拆除），并触发地面渲染项更新。
- `setEntity(col: number, row: number, entity: Entity | null): void`  
  更新格子上的实体占用（由 EntityManager 调用）。

#### 坐标转换
- `gridToWorld(col: number, row: number): { x: number, z: number }`  
  返回格子中心的世界坐标（XZ 平面，Y=0）。公式：
  ```typescript
  const x = (col - row) * (tileW / 2);
  const z = (col + row) * (tileH / 2);
  return { x, z };
  ```
- `worldToGrid(x: number, z: number): { col: number, row: number }`  
  返回世界坐标对应的网格坐标（四舍五入取整）。公式：
  ```typescript
  const col = (x / (tileW/2) + z / (tileH/2)) / 2;
  const row = (z / (tileH/2) - x / (tileW/2)) / 2;
  return { col: Math.round(col), row: Math.round(row) };
  ```

#### 高度查询
- `getGroundHeight(col: number, row: number): number`  
  返回指定格子的地面 Y 坐标（基于 `height` 属性）。
- `getGroundHeightAtWorld(x: number, z: number): number`  
  根据世界坐标计算地面高度（通过 `worldToGrid` 获取格子后返回高度）。

#### 可行走性
- `isWalkable(col: number, row: number): boolean`  
  返回格子是否可行走（需考虑 `walkable` 和 `entity` 是否为 null）。

#### 地面渲染
- `buildGroundRenderItems(projection: Projection, camera: IsoCamera): TileRenderItem[]`  
  遍历所有格子，计算每个格子的屏幕位置和深度，生成 `TileRenderItem` 列表。深度 = `projection.worldToScreen(x, z, 0).sy`（投影后的屏幕 Y 坐标）。
- `updateTileRenderItem(col: number, row: number): void`  
  当某个格子地形改变时，重新生成对应的 `TileRenderItem`，通知渲染器更新。

#### 辅助方法
- `getAllTiles(): TileData[][]`  
  返回完整地图数据（用于存档）。
- `getNeighbors(col: number, row: number): {col, row}[]`  
  返回上下左右四个方向的相邻格子（边界内）。

### 4. 实现要点

#### 4.1 坐标转换精度
- 世界坐标与网格坐标转换采用浮点运算，最终取整。注意边界情况（如点击格子边缘时可能误判）。
- 提供 `worldToGridExact` 方法用于精确判断（如拾取时可选最近格子）。

#### 4.2 地面渲染优化
- 将 `TileRenderItem` 缓存为静态列表，只在初始化或地形变化时重建。
- 深度值在每次相机移动或缩放时不会改变（投影不变），因此排序时深度是静态的，可直接使用。
- 如果地面图块数量大，可考虑使用离屏 Canvas 预渲染整个地面，每帧只绘制一张图（但需处理相机平移和缩放时的重绘）。

#### 4.3 实体占用的双向同步
- 当实体移动时，`EntityManager` 会调用 `setEntity(oldCol, oldRow, null)` 和 `setEntity(newCol, newRow, entity)`。
- `GridSystem` 负责更新 `TileData.entity`，并可通过事件总线发布 `tileOccupancyChanged` 事件，供其他模块（如寻路、UI）监听。

#### 4.4 动态地形
- 建造/拆除建筑时，需要更新 `TileData` 的 `type`、`walkable`，并重新生成对应的 `TileRenderItem`。
- 同时可能需要更新周围格子的高度（如梯田效果），需调用 `updateTileRenderItem` 刷新。

### 5. 模块协作
- **Projection**：用于屏幕坐标计算。
- **IsoCamera**：提供相机偏移和缩放（渲染项生成时需要相机参数，但深度计算时只依赖投影）。
- **EntityManager**：通过 `setEntity` 同步占用。
- **ResourceManager**：提供地形图块图片资源。
- **EventBus**：发布 `tileChanged`、`tileOccupancyChanged` 等事件。

## 二、 EntityManager（实体管理器）

### 1. 职责
- 管理所有动态和静态实体（角色、建筑、物品等）的生命周期。
- 负责实体的创建、销毁、查找。
- 每帧更新所有实体的逻辑（移动、动画等）。
- 为渲染器提供实体的渲染项（RenderItem），并实时计算深度。
- 同步实体与网格系统的占用关系。
- 提供实体间碰撞检测（可选，基于网格）。

### 2. 关键数据结构

```typescript
// 实体基类（抽象）
abstract class Entity implements RenderItem {
    public id: string;                 // 唯一标识
    public col: number;                // 网格列
    public row: number;                // 网格行
    public height: number;             // 实体自身高度（世界单位，用于 Y 偏移和深度）
    public spriteKey: string;          // 资源 key（用于获取图片）
    public frame: number;              // 当前动画帧索引
    public depth: number;              // 实时计算的深度值（每帧更新）
    
    constructor(id: string, col: number, row: number);
    public abstract draw(ctx: CanvasRenderingContext2D): void;   // 由子类实现绘制逻辑
    public update(delta: number): void;                           // 由子类实现更新逻辑
    public getWorldPosition(gridSystem: GridSystem): { x: number, z: number, y: number };
    public getScreenPosition(projection: Projection, camera: IsoCamera): { sx: number, sy: number };
}

// 实体管理器
class EntityManager {
    private entities: Map<string, Entity>;           // 实体 ID -> 实体实例
    private gridSystem: GridSystem;                   // 网格系统引用
    private projection: Projection;                   // 投影引用（用于深度计算）
    private camera: IsoCamera;                        // 相机引用（用于深度计算）
    private dynamicGroup: Entity[];                   // 所有实体的列表（便于遍历）
}
```

### 3. 主要方法

#### 实体生命周期
- `addEntity(entity: Entity): void`  
  添加实体，将其视觉对象加入动态组，并更新 `GridSystem` 的占用。
- `removeEntity(id: string): void`  
  移除实体，清理资源，更新 `GridSystem` 占用。
- `getEntity(id: string): Entity | undefined`
- `getEntitiesAt(col: number, row: number): Entity[]`  
  返回指定格子上的所有实体（可能有多个，如站在建筑上的人）。

#### 更新与渲染
- `updateAll(delta: number): void`  
  遍历所有实体，调用 `update(delta)` 更新逻辑。同时每帧重新计算每个实体的深度值（`depth = projection.worldToScreen(x, z, y).sy`）。
- `getRenderItems(): RenderItem[]`  
  返回所有实体的渲染项列表，用于 `CanvasRenderer` 收集。深度已在 `updateAll` 中计算好。
- `syncEntityPosition(entity: Entity): void`  
  当实体网格坐标变化时，更新其在 `GridSystem` 中的占用，并重新计算世界坐标和屏幕位置。

#### 辅助查询
- `getPlayer(): Entity | undefined`  
  返回玩家实体（如果约定玩家 ID 为 `'player'`）。
- `getAllEntities(): Entity[]`

### 4. 实现要点

#### 4.1 实体移动逻辑
- 移动时，实体通常需要调用 `setPosition(col, row)` 方法，该方法会：
  1. 调用 `gridSystem.setEntity(oldCol, oldRow, null)`
  2. 更新实体的 `col`、`row`
  3. 调用 `gridSystem.setEntity(newCol, newRow, this)`
  4. 通过 `syncEntityPosition` 更新屏幕位置（如果后续需要，可重新计算深度）
- 移动路径可由 PathFinder 提供。

#### 4.2 深度计算
- 深度值基于投影后的屏幕 Y 坐标，确保绘制顺序正确。公式：`depth = projection.worldToScreen(x, z, y).sy`。
- 对于建筑等高物体，`height` 属性影响 Y 坐标，从而影响深度，确保高物体绘制在低物体之后。

#### 4.3 动画系统
- 实体子类可管理动画状态机，在 `update` 中根据时间增量切换 `frame`。
- `draw` 方法根据 `frame` 从资源管理器中获取对应的子图（使用图集）。

#### 4.4 碰撞检测
- 基于网格的碰撞：移动前检查目标格子是否可行走（`gridSystem.isWalkable`）。
- 如果需要更精确的碰撞，可增加 `boundingRadius` 等属性，在移动时检查周围格子。

#### 4.5 渲染性能
- 实体渲染项列表每帧重建（因深度变化），但通常实体数量有限，开销可控。
- 可对静态实体（如建筑）单独处理，仅在位置变化时重新生成渲染项。

### 5. 模块协作
- **GridSystem**：用于获取地面高度、更新占用、检查可行走性。
- **Projection**、**IsoCamera**：用于计算屏幕位置和深度。
- **ResourceManager**：提供实体图片资源。
- **EventBus**：发布实体相关事件（如 `entityMoved`、`entityDied`），供 UI 和游戏逻辑监听。

## 三、 PathFinder（寻路系统）

### 1. 职责
- 基于网格地图，提供 A* 寻路算法。
- 支持地形代价（不同地形行走成本不同）。
- 支持动态障碍物（实体占用、临时障碍物）。
- 返回路径点（网格坐标数组）。

### 2. 关键数据结构

```typescript
// 路径节点
interface PathNode {
    col: number;
    row: number;
    g: number;      // 从起点到当前节点的实际代价
    h: number;      // 启发式估计代价（曼哈顿距离）
    f: number;      // g + h
    parent: PathNode | null;
}

// 寻路配置
interface PathConfig {
    allowDiagonal: boolean;     // 是否允许对角线移动（默认 false）
    terrainCost: Map<string, number>; // 地形类型 -> 移动代价（如草地1，沼泽2）
}
```

### 3. 主要方法

- `findPath(start: {col, row}, end: {col, row}, gridSystem: GridSystem, config?: PathConfig): {col, row}[]`  
  返回从起点到终点的路径点数组（不含起点），若不可达返回空数组。

#### 辅助方法
- `private heuristic(a: {col, row}, b: {col, row}): number`  
  启发函数，使用曼哈顿距离（`Math.abs(a.col - b.col) + Math.abs(a.row - b.row)`）。
- `private getNeighbors(node: PathNode, gridSystem: GridSystem, allowDiagonal: boolean): PathNode[]`  
  获取相邻可通行的节点。
- `private isWalkable(col: number, row: number, gridSystem: GridSystem, ignoreEntity?: Entity): boolean`  
  检查格子是否可行走（考虑地形 walkable 和实体占用，可忽略指定实体）。
- `private getMovementCost(col: number, row: number, gridSystem: GridSystem, config: PathConfig): number`  
  返回移动到此格子的代价（基于地形类型）。

### 4. 实现要点

#### 4.1 A* 算法核心
- 使用优先队列（二叉堆）存储 open 集，提高性能。
- 维护 closed 集（使用二维布尔数组或 Set 记录已处理节点）。
- 当从 open 集中取出节点时，如果该节点是终点，则回溯路径并返回。

#### 4.2 动态障碍物处理
- 每次寻路时实时查询 `gridSystem.isWalkable`，考虑实体占用和地形可行走性。
- 若允许对角线移动，需检查对角线两个相邻格子是否可行走，避免穿墙。

#### 4.3 地形代价
- 不同地形可配置不同代价，影响路径选择（如尽量走道路而非草地）。
- 代价在 `getMovementCost` 中获取，加到 `g` 值上。

#### 4.4 性能优化
- 对于频繁寻路（如多个单位），可缓存某些节点的启发式值。
- 若地图较大，可考虑使用跳跃点搜索（JPS）等优化，但本框架优先保证简单实现。

#### 4.5 边界检查
- 所有节点访问前需检查是否在地图范围内。

### 5. 模块协作
- **GridSystem**：提供地图数据（`isWalkable`、`getTile`）和地形类型。
- **EntityManager**：可提供忽略特定实体（如玩家自身）的选项，在 `isWalkable` 中判断。
- **EventBus**：可选，在路径计算失败时发布事件（如 `pathNotFound`）。
